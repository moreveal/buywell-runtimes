# История изменений

## 1.2.0 — 22 июля 2026

- Добавлен живой каталог товаров и их параметров для настройки сценария по стабильным ID.
- Покупка теперь передаёт обязательный ID товара, безопасное название из списка продаж, значения полей и отдельные ID выбранных вариантов.
- Выданный покупателю контент больше не может попасть в название или обычные данные события.
- HTTP-клиент получил раздельные таймауты и безопасные повторы только для операций чтения.

## 1.1.0 — 22 июля 2026

- Сценарий теперь может задать покупателю вопрос, проверить ответ и продолжить выполнение после получения корректного значения.
- Ожидания и ещё не обработанные ответы сохраняются после переподключения runtime.
- Установщик пытается восстановить отсутствующий `pip` через `ensurepip` и понятнее объясняет отсутствие поддержки виртуальных окружений.
- Добавлена автоматическая установка фоновой службы с текущим пользователем и папкой runtime для Linux и Windows.
- Установщик проверяет полноценное подключение runtime к Buywell, а повторная установка обновляет и перезапускает существующую службу без дубликатов.
- Инструкция по установке стала короче и больше не требует вручную составлять systemd-службу или задачу Windows.

## 1.0.1 — 22 июля 2026

- Runtime теперь поставляется одним ZIP-архивом со скриптами установки и запуска для Windows и Linux, конфигуратором, списком зависимостей и Python-кодом.

## 1.0.0 — 22 июля 2026

- Добавлены события новой покупки и сообщения покупателя.
- Добавлена отправка сообщения в текущий чат GGSel.
- Добавлены сохранение курсоров, дедупликация и повтор доставки событий.
- Добавлен автономный запуск на Windows и Linux.
