# FunPay Cardinal для Buywell

Модуль связывает FunPay Cardinal со сценариями Buywell. Он получает события об оплаченных заказах, изменениях статуса и сообщениях покупателей, а затем выполняет действия сценария в нужном чате.

## Что умеет модуль

- запускать сценарий после оплаты заказа или изменения его статуса;
- передавать в сценарий данные заказа, товара, покупателя, категории и отзыва;
- задавать покупателю вопросы и ждать корректный ответ;
- отправлять сообщения из сценария в исходный чат FunPay;
- продолжать ожидание ответа после переподключения Cardinal.

## Какие данные доступны сценарию

Поля с ID и поля с названиями разделены. Например, «ID категории» содержит стабильный идентификатор категории FunPay, а «Название категории» — текст, который видит пользователь. Для родительской категории доступны отдельные поля.

Статусы, валюты и типы категорий передаются понятными кодами: например, `paid`, `rub` и `common`. Внутренние числовые значения Cardinal не используются.

## Безопасность

Golden key и сессия FunPay остаются внутри Cardinal. Buywell не получает секреты автовыдачи, сообщение после оплаты, сырой HTML, CSRF-токены и приватные ссылки.

## Совместимость

Версия 1.2.2 рассчитана на FunPay Cardinal 0.1.17.8. Для подключения нужен исходящий доступ к `https://api.buywell.pro`.
